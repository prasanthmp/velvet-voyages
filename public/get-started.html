Features

What Wander includes

Work and Personal modes

Work mode prioritizes client-friendly, policy-aware choices. Personal mode uses a different theme and favors downtime fit.

Map and list views

Switch between a location-first map and a ranked list. Both update when filters change.

OpenTable restaurants

Restaurant recommendations are restricted to OpenTable-sourced entries and open directly to OpenTable search.

Google Maps directions

Directions use live browser location when available, otherwise the selected anchor location.

Preference questionnaire

Cuisine, ambience, allergies, price range, and expense limit shape recommendation scoring.

Calendar and CRM context

Sample calendar events set city, customer, purpose, party size, and time. CRM profiles add account tier and customer preference.

Expense policy fit

Policy profiles simulate Concur or travel-policy limits and flag recommendations as policy fit or policy review.

Availability scoring

Date, time, and party size simulate availability statuses: Available, Limited, and Waitlist.

AI fit score

Scores combine mode, distance, policy, cuisine, ambience, allergies, CRM context, availability, and feedback.

Why-this tags

Tags explain the ranking, such as policy-fit, crm-match, available, strategic, OpenTable, and party-size-fit.

Learning from feedback

Like and dislike buttons adjust future ranking on the current device using local storage.

Admin analytics

The analytics strip shows demo metrics for time saved, policy fit, accepted recommendations, and satisfaction.

Build Status

Production-ready versus hackathon-grade

Production-ready

Public HTTPS delivery, OCI Kubernetes hosting, static asset serving through Caddy, no-login dashboard access, responsive UI, seeded recommendation data, deterministic scoring, and browser-only state for feedback.

Hackathon-grade

Data is sampled instead of connected to live travel, HR, CRM, expense, OpenTable, or calendar APIs. AI scoring is implemented as explainable JavaScript logic rather than a trained model.

Next production step

Add enterprise identity, real policy systems, live reservation availability, privacy controls, audit logging, and feedback learning across devices.
